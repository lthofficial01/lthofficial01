# Project README Standard

프로젝트 유형이 달라도 첫 화면의 문법은 동일하게 유지합니다. 방문자가 20초 안에
`왜 했는지`, `무엇을 했는지`, `내가 맡은 일`, `결과`를 판단할 수 있어야 합니다.

## First-screen rules

1. 제목 아래에 프로젝트의 대상, 해결 방법, 핵심 결과를 1~2문장으로 요약합니다.
2. 핵심 기술 키워드는 5~8개만 노출합니다.
3. `프로젝트 개요` 표는 배경, 해결 과제, 담당 업무, 범위, 결과만 남깁니다.
4. 대표 이미지나 Demo가 있으면 첫 화면에 배치합니다.
5. 비어 있는 섹션과 장문의 기능 나열은 만들지 않습니다.

## 프로젝트 개요

| 구분 | 내용 |
| --- | --- |
| **배경** | 이 프로젝트가 필요한 이유를 한 문장으로 설명합니다. |
